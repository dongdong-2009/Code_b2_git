This is a windows program used to test AlarmStoreService.
It used AlarmListCtrl to display alarm detail and used corba call to communicate with AlarmStoreService.
The actions to AlarmStoreService include : new alarm, update alarm, sorting, filter and delete location alarms etc.

1) All alarms details are generated by a fonfiguration file. We can select it through GUI after it started up.
2) This program need command line parameters, we can use a .ini file to include all necessary parameters. 
Fox example: we can use the param: --run-param-file=./Configuration/AlarmStoreHarness.ini to run it. For details please refer to ./Configuration/AlarmStoreHarness.ini.
